## RED-state proof

Both new modules are imported lazily inside each test body so collection
does not crash on missing modules. Each test wears a
`@pytest.mark.xfail(strict=True)` marker citing the missing module
(`bonfire.engine.model_resolver` / `bonfire.engine.factory`). The Warrior
phase that lands `src/bonfire/engine/model_resolver.py` and
`src/bonfire/engine/factory.py` graduates the tests to XPASS, and Sage's
strict-xfail then turns those XPASSes into failures, forcing the Warrior
to remove the markers in the same diff. This is the canonical Bonfire
RED pattern — the marker is the contract, not the bug.

## Notes for the next Sage / Wizard pass

1. **Module path discrepancy in the dispatch brief.** The mission text
   states "engine.tiers is the existing module, not agent/tiers." The
   Sage memo §A and the on-disk source both place `resolve_model_for_role`
   in `bonfire.agent.tiers` (cluster-350 D-CL.1 ratified). I followed the
   Sage memo as the binding authority. If the Warrior or Sage finds a
   reason to add a re-export from `bonfire.engine`, the helper test #2
   still passes because the helper imports `resolve_model_for_role` from
   `bonfire.agent.tiers` per Sage §C.1.

2. **Parametrize + xfail count.** Sage §H.5 locks "10 tests" for Knight A
   counted by logical contract; pytest collects 12 items because §H.2.4
   is parametrized over three failure modes. If Sage prefers a
   single-item never-raise test, the parametrize block can be flattened
   to one body without changing the contract. I kept parametrize because
   §H.2.4's reason ("Locks the never-raise contract") reads stronger
   when the three exception families are independently witnessed.

3. **`caplog` vs stderr.** Sage §C.2 says "emits a stderr warning" but
   the example implementation uses `logger.warning(...)`. I assert
   against `caplog` records on the `bonfire.engine.factory` logger
   (canonical pytest pattern). Routing `logger.warning` to stderr is
   the default `logging` behaviour; the warning is observable on both
   surfaces. If the Warrior emits via `sys.stderr.write` instead, the
   tests will need a small reroute — flag in the Warrior memo.

4. **`monkeypatch.setattr(BonfireSettings, "__init__", ...)` in test 5
   and test 4 of factory.** This mutates a Pydantic model class for the
   duration of the test. `monkeypatch` restores after teardown. No
   global state leaks observed in the (unrun) test layout.

5. **Bash-denied posture.** This Knight session ran without Bash access.
   Files are written to disk in the worktree at relative paths per
   `feedback_worktree_relative_paths`. Wizard runs `pytest` + `ruff`,
   commits + pushes per the Agent Commit Protocol in `CLAUDE.md`.
