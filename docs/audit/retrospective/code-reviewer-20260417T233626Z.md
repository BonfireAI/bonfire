## Lens-by-Lens Assessment

### 1. Pydantic v2 idiom quality — PASS with inconsistencies
`ConfigDict` is used in envelope.py:70 and plan.py (all four models), but events.py:45 uses a plain dict `{"frozen": True}`, and envelope.py:37/46 declare `frozen=True` as a class kwarg. All three forms are legal; the inconsistency hurts readability. See R-002, R-003. `AliasChoices` and `populate_by_name=True` are correctly paired. `field_validator` and `model_validator` are used in idiomatic mode-before/mode-after positions. `TypeAdapter` usage in events.py:333 is correct.

### 2. Type-hint completeness — PASS
Modern forms throughout: `X | None`, `list[X]`, `dict[K, V]`, `Annotated[...]`, `Literal[...]`. No stale `typing.Optional/List/Dict`. `from __future__ import annotations` applied. One justified `# noqa: TC003` on `Path` in envelope.py:11 (Pydantic needs it at runtime) — correct per the "TC003 breaks Pydantic" discipline.

### 3. Error message quality — MIXED
config.py is disciplined: messages include the failing value (`f"budget must be non-negative (got {v})"`). envelope.py is not: `"cost_usd must be >= 0"` alone, no value, repeated in two places. See R-004. DAG validation messages are excellent — they include known-stage lists and cycle paths. "stages must not be empty" is minimal but adequate.

### 4. Cycle-detection algorithm brittleness — FAIL (R-001)
Coloring scheme is textbook-correct (white/gray/black DFS). Self-loop exclusion in adjacency (line 138) is defensive and fine. But the path-reconstruction logic at lines 150-157 produces strings with a duplicate terminal node — `a → b → b` for a two-node cycle, `c → b → a → a` for a three-node cycle. Reproduced empirically. Tests pass because they only assert node-name membership and arrow presence; the format is not checked. This is the single MAJOR finding — see R-001 for the fix. Also: no validation against duplicate entries in `depends_on`, which can produce malformed adjacency (R-006).

### 5. API shape + brittleness — PASS with minor notes
`event_type.split(".")[0]` is safe because Literal types enforce the format. `_CURRENT_SCHEMA_VERSION = 4` is a magic number with no enforcement — `config_version` is accepted from TOML without comparison (R-007). `__repr__` formatting (`cost=$0.1234`) matches the private V1 contract — correct for a transfer-target ticket.

### 6. Dead code / unused imports — PASS
ruff check passes on src/ and tests/. No semantic dead code spotted.

### 7. Error handling — PASS
No broad `except:` clauses. `_migrate_legacy_keys` silently skips non-dict sections — intentional and documented. Validators raise `ValueError` with specific messages (see R-004 for an improvement).

### 8. Maintainability — PASS with one footgun (R-005)
`Envelope.chain(parent, **overrides)` uses `defaults.update(overrides)`, allowing a caller to overwrite `envelope_id` or `parent_id` and break the chain invariant. Fidelity to private V1 likely preserves this; flagging as a latent footgun, not a blocker. Naming is clean; docstrings scrubbed of private refs evaluate well on their own merit.
