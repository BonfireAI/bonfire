## Summary

The Warrior implementation is V1-verbatim and engineering-sound. All eight focus areas pass:

1. Async patterns — `_attempt_once` awaits backend under `asyncio.timeout`; SDK backend wraps the async generator in `aclosing()` for clean early-return. No sync/async mixing.
2. Consumer/handler isolation — dispatch emits to an optional EventBus; EventBus itself owns per-handler try/except with `logger.exception` (bus.py:108-129). Dispatch does not own handlers; the division of responsibility is correct.
3. Sequence stamping — EventBus stamps via `model_copy(update={"sequence": n})`; dispatch emits with `sequence=0` as a sentinel the bus overrides. Frozen-model-safe.
4. Pydantic v2 idioms — `DispatchResult` uses `ConfigDict(frozen=True)`, bare annotations, runtime `Envelope` import with correct `TC001 noqa`.
5. Retry algorithm — `delay = retry_delay * (2 ** attempt)`, no pre-first-attempt sleep, no post-exhaustion sleep, `retries` counts retries, exhaustion total calls = `max_retries + 1`. All cardinality variants covered by tests.
6. SDK backend error wrapping — outer try/except wraps `_do_execute`; ErrorDetail populated with `error_type`, `message`, `traceback`. Rate-limit and agent-error paths return error envelopes, not raise.
7. Pydantic AI lazy import — module-level `Agent: Any = None`; lazy `from pydantic_ai import Agent as _Agent` inside `execute()`; module loads without pydantic_ai installed.
8. TierGate — genuinely stateless, returns `True` for all inputs; unicode/whitespace/long-string matrix covered.

### ENG-01 (low) — check_tier kw-only docstring vs impl

`tests/unit/test_dispatch_tier.py:15` says:

```
* ``check_tier(*, agent_name, model, tier="free") -> bool`` — keyword-
```

but `src/bonfire/dispatch/tier.py:12-23` and Sage §3 both declare it positional-or-keyword. No test actually verifies TypeError on positional call; every call site in the suite uses keyword form anyway. Doc-only drift. Fix in a follow-up either by adding `*,` in impl + a negative test, or by aligning the test docstring to the impl+Sage spec.

### ENG-02 (low) — exception-exhaustion overwrites earlier error_type

`runner.py:222-234` rebuilds the final envelope with `ErrorDetail(error_type='infrastructure', message=str(exc))`. If an earlier attempt produced a FAILED envelope with a richer (non-terminal, retryable) `error_type`, that detail is not surfaced. This is locked by Sage §6 and is V1-verbatim, so it's correct-by-spec, but operators diagnosing mixed failure modes will only see the final exception. Non-blocking; worth a follow-up ticket.

### ENG-03 (info) — _emit does not protect dispatch from bus.emit() raising

`_emit` at `runner.py:50-53` calls `await bus.emit(event)` unguarded. Today's EventBus never re-raises from handler failures (bus.py catches per-handler), but if a future bus refactor leaked an exception out of `emit()` itself, it would hit the `except Exception` block at runner.py:201 and be misattributed as an infrastructure failure, falsely incrementing retries. Coupling observation only — no action today.
