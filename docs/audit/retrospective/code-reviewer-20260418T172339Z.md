### CR-02 — [suggestion] `get()` raises bare `KeyError`

**File:** `src/bonfire/workflows/registry.py:40-50`

`get()` raises `KeyError` with a human-readable message. Matches V1 and Sage lock. A dedicated `UnknownWorkflowError(KeyError)` subclass would let callers catch specifically without message pattern-matching, but it is beyond transfer scope and V1 does not do it.

**Recommendation:** Defer. File a follow-up only if a consumer actually needs the distinction.

The frozen-mutation test uses `pytest.raises(Exception)` to accept `ValidationError`. Breadth is deliberate: robust against Pydantic renaming its error class.

**Recommendation:** Accept as-is.

### CR-04 — [suggestion] Local imports inside `get_default_registry`

**File:** `src/bonfire/workflows/registry.py:67-78`

Local (function-scope) imports of the factories. Matches V1. Not a cycle-breaker in practice (factories don't import registry), but keeps the module factory-agnostic at load time.

**Recommendation:** Optional one-line comment to document intent.

### CR-05 — [suggestion] `TestDependencyConstraints` uses `vars()` scan

**File:** `tests/unit/test_workflows.py:1370-1385`

**Recommendation:** No change.

**File:** `src/bonfire/workflows/research.py:85-89`
