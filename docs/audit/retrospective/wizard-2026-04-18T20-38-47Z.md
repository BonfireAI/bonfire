## Findings

### W-01 — LOW — Scrub leak in test-file docstrings

All 7 canonical `test_engine_*.py` files open with a provenance docstring:

```python
"""Canonical RED — ``bonfire.engine.X`` (BON-334).
```

Sage decisions doc (line 17-18) declares: *"no BON-\d+ anywhere in the canonical suite"*.
The prompt's three-adaptations §1 likewise says *"zero BON-NNN"*. The merged baseline
(`test_envelope.py`, `test_event_bus.py`) already carries identical patterns from prior
waves BON-331, BON-333. So this is **consistent with merged baseline**, not a PR-specific
regression — but technically violates Sage D's wording.

**Recommendation:** Merge as-is. Spin a cleanup follow-up to scrub provenance tags
from test docstrings across ALL engine-era test files at once if strict scrub is
desired. Not a merge blocker.

### W-02 — INFO — Hardening deviation not listed in prompt's six

Public wraps `handler.handle()` (pipeline.py:476-484) and `_execute_single_inner`
(executor.py:96-108) in try/except that build ErrorDetail envelopes. V1 pipeline.py:425
invokes `handler.handle()` bare. Public is strictly safer — a handler raise now fails
only the one stage rather than exploding into `run()`'s outer catch. The prompt lists
six Warrior deviations; this is a seventh that is a clean hardening consistent with
C19 ("execute_single never raises — exceptions become failed Envelopes").

**Recommendation:** Note in next handoff as deviation #7 (hardening). No code change.
