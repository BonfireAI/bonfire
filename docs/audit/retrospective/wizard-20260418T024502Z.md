### 5. Sequence stamping — PASS

```python

```

Stamping happens exactly once per `emit`, before any handler runs. `model_copy` on a frozen Pydantic model produces a new instance — original untouched.

Tests (8): first emit = 1, successive increment, counter global across types, caller's sequence overwritten, original not mutated, handler receives copy (`captured[0] is not original`), all handlers receive same stamp, other fields preserved (including `event_id` identity after copy).

Edge case `test_emit_with_no_subscribers_still_increments_sequence` locks the invariant even when no handler observes the stamp. Good.

## Findings

### W-001 — nit, documentation fidelity

**File:** `src/bonfire/events/bus.py` line 73
**Summary:** Docstring cites `type(event) is event_type` but the impl uses `self._typed.get(type(event), [])`.

The behavior is identical (dict hashing on type objects is identity-based), so the contract holds. The literal expression cited in the docstring does not appear in the code. Matches V1 verbatim; V1-inherited micro-nit.

**Suggestion:** Optional post-merge — rephrase docstring to "dispatched via dict lookup on `type(event)`, which gives exact-type semantics (subclasses do not trigger parent handlers)." No code change.

### W-002 — info, reentrancy semantics

**File:** `src/bonfire/events/consumers/cost.py` line 47
**Summary:** `CostTracker._on_dispatch_completed` re-enters `bus.emit` inside a handler; outer and nested sequence numbers interleave.

When cost crosses 80% or 100%, the tracker awaits `self._bus.emit(CostBudgetWarning/Exceeded)` while the outer `emit(DispatchCompleted)` is still iterating its typed loop. The nested emit increments `self._sequence`, so a later typed subscriber for the outer event sees it stamped with `N` while the nested Warning carries `N+1`.

This is CORRECT by design — the outer stamp is frozen at outer-emit entry (`stamped = event.model_copy(update={"sequence": self._sequence})` on line 105) and is not re-assigned. Matches V1.

No test explicitly asserts the stamp-invariance-across-nested-emit contract. The suite covers externally visible behavior adequately, so this is informational only — not a gap requiring a test.

**Suggestion:** Consider a future doc note in `bus.py` acknowledging that handlers may re-enter `emit` and that the outer event's stamp is invariant across the handler's lifetime.
