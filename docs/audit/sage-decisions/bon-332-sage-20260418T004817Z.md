**Follow-up note for Warrior / future design:** If product direction later wants to tighten `cognitive_mode` to a Literal alphabet, that is a **breaking schema change** and must flow through an ADR + migration. Do not speculatively tighten in this transfer.

### 6. Shim + TYPE_CHECKING placement

**Decision:** TYPE_CHECKING block appears BEFORE the runtime import shim (Knight-A pattern). Rationale: Python parses and typechecks the TYPE_CHECKING names against the `if TYPE_CHECKING:` branch, and ruff sees the noqa markers cleanly. Knight-B's post-shim placement also works but creates a subtle ordering dependency if the shim itself ever wants TYPE_CHECKING-only names.

**`noqa: F401` on TYPE_CHECKING imports** — load-bearing: ruff cannot see that these names are used via deferred stringified annotations inside the conformer test classes.
