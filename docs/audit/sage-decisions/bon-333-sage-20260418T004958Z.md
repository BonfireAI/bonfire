## Structural divergences

### 1. Snapshot vs live iteration on mid-emit subscription (LOAD-BEARING)

**Knight-A claim:** `test_subscribe_registers_new_handler_only_seen_by_next_emit` asserts that a handler registered **during** a handler's body does NOT fire for the in-flight emit (snapshot semantics).

**V1 evidence — `bonfire/src/bonfire/events/bus.py:108`:**
```python
for handler in self._typed.get(type(event), []):
```
This returns a **live reference** to the per-type list. Python's `for` loop over a list is not snapshot-safe: appending to the list during iteration can cause the appended element to be visited within the same loop iteration sequence.

**V1 docstring (lines 1–27):** Mentions "Consumer isolation", "Sequential handler execution", "Ordering guarantee (C7)", "Sequence stamping". There is ZERO mention of snapshot semantics, registration-list copy, or mid-emit subscription behavior.

**V1 tests (`bonfire/tests/unit/test_event_bus.py`, 616 LOC):** Does NOT contain any mid-emit-subscribe assertion. The behavior is undefined and untested in V1.

**Sage decision: DROP snapshot assertion.** V1 is live iteration. Asserting snapshot would force the Warrior to deviate from V1 (`list(self._typed.get(...))` or similar copy). That is out-of-scope for W2.3, which is a **transfer**, not a redesign. The canonical suite does NOT assert behavior on mid-emit subscription — the contract is silent on that path, matching V1.

No follow-up ticket filed. If a future consumer depends on snapshot semantics, that ticket owns the design decision (and the `list(...)` copy).

### 2. `VaultIngestConsumer` contract

**Knight-A claim:** `VaultIngestConsumer` is a namespace re-export. Test surface only (importable, has `register` method).

**Knight-B claim:** Full class contract — `VaultIngestConsumer(backend=, project_name=)`, subscribes to 4 event types (`StageCompleted`, `StageFailed`, `DispatchFailed`, `SessionEnded`), stores entries on the backend, resilience on backend failure.

**V1 evidence — `bonfire/src/bonfire/events/consumers/vault_ingest.py` (5 LOC):**

**Public v0.1 state:** `bonfire/vault/` does NOT exist. `bonfire/protocols` does NOT exist. None of the downstream machinery is present.

**Sage decision — hybrid:** Test the **public surface** that W2.3 is responsible for:
1. `VaultIngestConsumer` is importable from `bonfire.events.consumers.vault_ingest`.
2. Constructor accepts `backend=, project_name=`.
3. Has a `register(bus)` method.
4. After `register(bus)`, the bus has typed subscriptions for the four event types: `{StageCompleted, StageFailed, DispatchFailed, SessionEnded}`.

**Explicitly OUT of W2.3 scope** (deferred to whatever wave transfers `bonfire.vault/`):
- Content hashing / dedup semantics.
- Backend.store interaction details.
- Entry content format / metadata structure.
- Backend-failure resilience internals.

**Warrior options for GREEN:**
- **Option A (preferred):** Implement a minimal `VaultIngestConsumer` class inline in `src/bonfire/events/consumers/vault_ingest.py` that stores the backend/project_name, defines `register(bus)` with the four subscriptions, and stub handlers that call `backend.store(...)` if available (or no-op). No hashing, no dedup, no protocols import.
- **Option B:** Re-export path — create a stub `bonfire.vault.consumer` module with the minimal class, match V1's re-export shape. Larger footprint, pulls forward scope.

The canonical tests are **Option-A-friendly** but pass for either approach, as long as the surface contract holds.

## Locked signatures (Warrior handoff)

### `bonfire.events.bus`

```python
E = TypeVar("E", bound=BonfireEvent)

class EventBus:
    def __init__(self) -> None: ...
    def subscribe(
        self,
        event_type: type[E],
        handler: Callable[[E], Awaitable[None]],
    ) -> None: ...
    def subscribe_all(
        self,
        handler: Callable[[BonfireEvent], Awaitable[None]],
    ) -> None: ...
    async def emit(self, event: BonfireEvent) -> None: ...
```

- Sequence counter: starts at 0, **first emit stamps 1**, monotonic across all event types.
- Stamping via `event.model_copy(update={"sequence": n})` — original frozen, never mutated.
- Exact-type filter: `type(event) is event_type` — subclasses do NOT trigger parent-type handlers.
- C7 ordering: typed handlers fire BEFORE global handlers, regardless of registration order.
- Within each tier: registration order.
- Sequential await: all handlers awaited one at a time (not `asyncio.gather`).
- Consumer isolation: each handler wrapped in `try/except`; exceptions logged, never re-raised.
- Counter advances even when no subscribers / handler fails.

### `bonfire.events.__init__` (re-exports)

```python
from bonfire.events.bus import EventBus
from bonfire.models.events import BonfireEvent
__all__ = ["BonfireEvent", "EventBus"]
```

### `bonfire.events.consumers.logger`

```python
class SessionLoggerConsumer:
    def __init__(self, persistence: Any) -> None: ...
    async def on_event(self, event: BonfireEvent) -> None: ...
    def register(self, bus: EventBus) -> None: ...  # uses bus.subscribe_all
```

- `on_event` calls `persistence.append_event(event.session_id, event)` — **positional args, in that order**.
- Exceptions from `append_event` are caught and logged (warning level). `on_event` NEVER raises.

### `bonfire.events.consumers.display`

```python
class DisplayConsumer:
    def __init__(
        self,
        callback: Callable[..., Any],
        persona: Any | None = None,  # optional — canonical tests do NOT exercise persona branch
    ) -> None: ...
    def register(self, bus: EventBus) -> None: ...
```

- Subscribes to exactly: `{StageCompleted, StageFailed, QualityFailed, CostBudgetWarning}`.
- Detects sync vs async callback via `asyncio.iscoroutinefunction(callback)`.
- Callback exceptions are caught and logged; consumer is isolation-safe.
- `StageCompleted` → message contains `stage_name`, `duration_seconds`, `cost_usd`.
- `StageFailed` → message contains `stage_name`, `error_message`, substring `"FAILED"` (case-insensitive).
- `QualityFailed` → message contains `gate_name`, `message`.
- `CostBudgetWarning` → message contains the `percent` as a substring.

### `bonfire.events.consumers.cost`

```python
class CostTracker:
    def __init__(self, budget_usd: float, bus: EventBus) -> None: ...
    @property
    def total_cost_usd(self) -> float: ...
    def register(self, bus: EventBus) -> None: ...  # subscribes to DispatchCompleted
```

- Accumulates from `DispatchCompleted.cost_usd`.
- At `>= 80%` budget: emit `CostBudgetWarning(..., percent=...)` once.
- At `>= 100%` budget: emit `CostBudgetExceeded(...)` once.
- Both thresholds are latched — no repeat emissions.
- Ignores all non-`DispatchCompleted` events.

### `bonfire.events.consumers.vault_ingest`

```python
class VaultIngestConsumer:
    def __init__(self, backend: Any, project_name: str) -> None: ...
    def register(self, bus: EventBus) -> None: ...
```

- After `register`: bus has typed subscriptions for `{StageCompleted, StageFailed, DispatchFailed, SessionEnded}`.
- Handler internals (hashing, store calls, metadata) are **out of W2.3 scope** — only the surface + subscription set are tested.

### `bonfire.events.consumers.__init__` — `wire_consumers`

```python
def wire_consumers(
    *,
    bus: EventBus,
    persistence: Any,
    cost_tracker: CostTracker,
    display_callback: Callable[..., Any],
    vault_backend: Any,
) -> None: ...
```

- Keyword-only.
- After calling: `StageCompleted` → `display_callback` is called AND `persistence.append_event` is called.
- After calling: `DispatchCompleted` → `cost_tracker.total_cost_usd` accumulates.
- `persona` / `cost_ledger_path` kwargs are deliberately omitted from canonical tests (future-proof — Warrior MAY add them, but the locked signature is the five params above).
