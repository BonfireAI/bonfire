## 1. Scope decisions — the five questions

### 1.1 `pydantic_ai_backend` inclusion — **KEEP**

**V1 evidence:** `src/bonfire/dispatch/pydantic_ai_backend.py` (77 LOC).
The backend uses a lazy-import pattern — module-level `Agent: Any = None`
with the real `from pydantic_ai import Agent as _Agent` done INSIDE
`execute()` on first call. The module loads cleanly whether or not
`pydantic_ai` is installed.

`pyproject.toml` for `bonfire-public` does NOT list `pydantic_ai` as a
runtime dependency, which is fine: tests patch
`bonfire.dispatch.pydantic_ai_backend.Agent` directly, so no real import
is triggered. Sealing the `AgentBackend` protocol as truly provider-
agnostic is the core reason this backend ships in public v0.1 despite
being light.

Both Knights covered it (Knight-A: 16 tests; Knight-B: 7 tests). Sage
ships 15 canonical tests.

### 1.2 `dispatch_wiring` tests — **CUT**

**V1 evidence:** `tests/unit/test_dispatch_wiring.py` targets
`bonfire.cli.commands.dispatch` (an XP composition root). Every test
inspects the source of that module for `XPTracker`, `XPCalculator`,
`XPConsumer`, `wire_consumers`, and `get_default_handlers` — none of
which exist in public v0.1.

The public CLI is flat (`src/bonfire/cli.py`), there is no
`bonfire.cli.commands` package, and the XP system is out of W3.2 scope.
Knight-B was correct to cut these tests. Sage confirms the cut.

### 1.3 `resolve_llm_backend` function — **OUT OF DISPATCH SCOPE**

**V1 evidence:** grep for `resolve_llm_backend` / `def resolve_.*backend`
in `/home/ishtar/Projects/bonfire/src/` returns ZERO hits inside
`src/bonfire/dispatch/`. The function lives in

Both Knights correctly omitted it. Dispatch callers instantiate
`ClaudeSDKBackend()` / `PydanticAIBackend()` directly.

### 1.4 Jitter in backoff — **DETERMINISTIC, NO JITTER**

**V1 evidence:** grep for `jitter|random\.|random\(` in
`/home/ishtar/Projects/bonfire/src/bonfire/dispatch/` returns ZERO
matches. The runner implements `delay = retry_delay * (2 ** attempt)`
(runner.py line 159 and 216). Sage locks deterministic backoff in the
test suite (timing lower-bound asserts with safe margins for CI).

### 1.5 Budget exhaustion mid-retry — **NOT RUNNER'S JOB**

**V1 evidence:** `runner.py` reads no budget counter between attempts.
`DispatchOptions.max_budget_usd` is passed through to the backend's
`ClaudeAgentOptions` so the SDK enforces per-call budget; there is no
cross-attempt budget short-circuit in the runner loop. Budget
enforcement at the pipeline level lives in the engine, not dispatch.
Sage does not add tests for this in W3.2.

## 2. Structural decisions

### 2.1 File count — **5 files** (Knight-A layout)

`test_dispatch_result.py` is split from `test_dispatch_runner.py` because
`DispatchResult` carries substantial frozen-model / serialization /
type-drift coverage (20 tests in Knight-A) that belongs with the model,
not the runner. Knight-B's consolidated runner file covered the shape
lightly (5 tests); splitting aligns with the public repo's module layout
(`result.py` is a separate file) and keeps runner tests focused on
retry semantics.

### 2.2 Import shim — per-file, matching `tests/unit/test_envelope.py`

Every file has:

```python

    _IMPORT_ERROR = _exc
```

Collection succeeds; every test fails with the import error until the
Warrior lands the module. Matches public repo convention.

### 2.3 Package re-exports

`bonfire.dispatch.__all__` must contain `{"DispatchResult", "TierGate",
"execute_with_retry"}` and each symbol must be the same object as the
canonical module-level export. Tests in `test_dispatch_result.py`,
`test_dispatch_tier.py`, and `test_dispatch_runner.py` all verify
`pkg_cls is mod_cls` (object identity, not just importability).

### 2.4 asyncio

`pyproject.toml` has `asyncio_mode = "auto"` — async test methods are
auto-marked. Sage follows Knight-B's cleaner style (no `@pytest.mark.asyncio`
decorator).

### 3.2 Retry semantics

`retries` counts RETRIES (not attempts). First-attempt success →
`retries == 0`. Two retries then success → `retries == 2`. Exhaustion
with `max_retries=3` → `retries == 3`. Total call count on exhaustion =
`max_retries + 1`.

### 3.3 Event shape

Events are imported from `bonfire.models.events`:

- `DispatchStarted(session_id, sequence, agent_name, model)`
- `DispatchCompleted(session_id, sequence, agent_name, cost_usd, duration_seconds)`
- `DispatchFailed(session_id, sequence, agent_name, error_message)`
- `DispatchRetry(session_id, sequence, agent_name, attempt, reason)`

`sequence` is stamped at emission by the bus (0 from the runner, bus
increments — per BON-333). `session_id` equals `envelope.envelope_id`.
`attempt` in retry events is **1-indexed** (first retry = 1).

## 6. Warrior handoff summary

- **Module layout:** `src/bonfire/dispatch/{__init__,runner,tier,result,sdk_backend,pydantic_ai_backend}.py`.
- **Package surface:** `__all__ = ["DispatchResult", "TierGate", "execute_with_retry"]`.
- **Retry algorithm:** exponential, deterministic, `delay = retry_delay * 2 ** attempt_index`; no sleep before first attempt; no sleep after final exhausted attempt.
- **Terminal errors:** frozenset literal above — return immediately, no retry.
- **Exception-only exhaustion:** build final envelope via `envelope.with_error(ErrorDetail(error_type="infrastructure", message=str(exc)))`.
- **Cost accumulation:** `cumulative_cost += result_env.cost_usd` after each backend call that RETURNED (not after exceptions).
- **Event emission:** one `DispatchStarted` at the top; one terminal (`DispatchCompleted` XOR `DispatchFailed`); N `DispatchRetry` events where N equals the `retries` field on the returned result; `attempt` field on retry events is 1-indexed.
- **No jitter. No budget short-circuit. No resolver function in `dispatch/`.**
