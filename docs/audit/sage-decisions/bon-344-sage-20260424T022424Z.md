## Appendix — v1 surprises (scout notes)

1. **Bus `_typed` attribute access.** v1 test `test_subscribes_to_pipeline_completed` directly accesses `bus._typed[PipelineCompleted]` (private). v0.1's EventBus exposes `_typed` via the same `defaultdict`. Test will port cleanly but the leaky private-access pattern is worth noting for a future "BusInspection protocol" refactor. Not in this ticket scope.
2. **PipelineFailed synthesis pattern.** v1 `XPConsumer._handle_pipeline_failed` synthesizes a `PipelineCompleted`-shaped event with zero cost / zero duration / zero stages, then delegates. This is a regression-pattern from Session 033 ("XP-shows-0 bug"): the v1 tests (`TestBusHandlerDelegation`) were written specifically to prove bus.emit actually drives the handler. Port this pattern verbatim — do NOT "clean up" the synthesis in favor of a parallel handler path.
3. **`record()` has an unused `respawn` param in write-path.** v1 `XPTracker.record(xp_total, success, respawn=False)` writes `respawn` to JSONL but the consumer call path (`self._tracker.record(result.xp_total, success, result.respawn)`) always uses the calculator's result.respawn — so the JSONL always reflects the computed respawn state. Correct, just redundant. Port verbatim.
4. **Temperature floor math.** v1 uses `int(temp)` — fractional temperatures truncate down. At day 6.9 with `_TEMP_MAX=100, _TEMP_FLOOR=10, _DECAY_DAYS=7`: `100 - (90 * 6.9 / 7) = 100 - 88.71 = 11.29 → int → 11`. At day 7 exactly: the `if days_idle >= _DECAY_DAYS: return _TEMP_FLOOR` early-return fires. No test exercises day 6.9; port verbatim.
5. **`XPDisplayConsumer._select_phrase` returns non-empty string only.** Even with no phrase pool, the fallback always produces a string. The dash-separated output in `render_xp_awarded` is therefore always `+N XP — Awarded` (not `+N XP — ` with trailing empty). Test `test_fallback_produces_readable_text` asserts this: `"— " in output[0] and output[0].strip().endswith("Awarded")`. Port verbatim; do not "simplify" to return empty string on fallback (that would break the dash-rendering contract).
6. **Respawn `checkpoint=""` on emit.** v1 always emits `XPRespawn(... checkpoint="", ...)` — the actual checkpoint path is not wired through. This is a known gap (Session 033 era). Do NOT fill it in this ticket. Follow-up ticket to plumb checkpoint path through PipelineFailed payload (not scoped here).
7. **No v1 BON-NNN refs in the ported source.** All five v1 source files are clean of ticket-tracking strings (unlike some v1 test files). Docstring-strip in D5 is likely a no-op; still verify during port.
8. **No `XPAwarded.level_up` field.** The level-up signal is smuggled through the `reason` string (e.g. `"Pipeline completed — leveled up to Level 2: Flame"`). v1 test `test_level_up_detected` asserts `"level" in reason.lower()` and `"Flame" in reason`. This is ugly but shipped; port verbatim. A proper level-up field on `XPAwarded` (with `new_level: str | None = None`) is future work (Ticket B adjacent).
