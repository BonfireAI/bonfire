## Per-innovation verdicts

### Innovation 1 — `TestModelTierByteEquality` — ADOPT

**Sage memo cite:** §D3 ("Values are lowercase, ASCII, single-word, no separators") + the locked package surface block at memo §D2 lines 127-129 (`REASONING = "reasoning"`, `FAST = "fast"`, `BALANCED = "balanced"`).

**v0.1 verification:** Memo locks the literal byte values; this innovation pins them at the encoded-bytes level (`len(...)` and `.encode("ascii")`). Catches `"Reasoning"` (capitalization), `"reasoning "` (trailing space), or `"reasoñing"` (non-ASCII).

**Adversarial check:** Floor's `test_reasoning_value` asserts `== "reasoning"`, but StrEnum's coercion would let a typo'd `REASONING = "reasoning "` (trailing space) silently pass on `.value == "reasoning"`-style asserts in some refactor paths. The byte-encoding guard catches that.

**Reason for ADOPT:** Cheap byte-level lock on the most contractually visible field of the enum.

### Innovation 2 — `TestModelTierStrEnumSemantics` — ADOPT

**Sage memo cite:** §D3 ("Type is StrEnum, not str | Literal[...], not a plain dataclass. Matches AgentRole precedent (src/bonfire/agent/roles.py)").

**v0.1 verification:** Confirmed `src/bonfire/agent/roles.py:15` reads `class AgentRole(StrEnum):`. The drift-guard pins the same StrEnum invariant for ModelTier.

**Adversarial check:** Asserts `isinstance(tier, str)` for every member + `set(ModelTier)` cardinality 3. The set-cardinality piece is mildly redundant with floor's `test_three_values_exist` (which uses `len(list(...))`), but the str-isinstance check is genuinely new and catches a refactor to plain `Enum`.

**Reason for ADOPT:** Locks the StrEnum precedent; cheap and orthogonal to the floor.

### Innovation 3 — `TestModelTierJsonSerializable` — ADOPT

**Sage memo cite:** §D3 ("Values are the canonical serialized form -- used in TOML, JSONL, CLI output, and grep patterns") at memo §D2 lines 122-125 of the locked `tiers.py` docstring.

**v0.1 verification:** No direct v0.1 source — this guards a future use site. Memo explicitly lists JSONL as a target serialization.

**Adversarial check:** `json.dumps({"tier": tier.value})` round-trip. StrEnum values are already plain `str`, so this round-trips trivially today; the test ratchets that property so a future "subclass with __slots__" or "dataclass wrapper" refactor breaks loudly.

**Reason for ADOPT:** Forward-compat serialization invariant for JSONL/TOML emission paths.

### Innovation 4 — `TestCrossVocabularyParametrize` (10 parametrized cases) — ADOPT

**Sage memo cite:** §D3 (the GAMIFIED_TO_GENERIC alias map at memo §D2 lines 137-147) + `src/bonfire/naming.py:40-50` (the source-of-truth professional<->gamified pairing).

Plus the §D-CL.1 prover→verifier alias.
