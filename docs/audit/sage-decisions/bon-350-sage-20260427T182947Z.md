### `src/bonfire/agent/tiers.py` — CREATED (~95 LOC)

```python
"""Model tier vocabulary and role-aware resolver — BON-350.

Three-tier capability axis (reasoning/fast/balanced) decoupled from the
commercial tier (free/pro/enterprise) handled by ``bonfire.dispatch.tier``.

The resolver is the public primitive consumed by BON-351's dispatch
integration. Pure synchronous function — no I/O, no async, no cache.
"""

from __future__ import annotations

from enum import StrEnum
from types import MappingProxyType
from typing import TYPE_CHECKING, Mapping

from bonfire.agent.roles import AgentRole

if TYPE_CHECKING:
    from bonfire.models.config import BonfireSettings

__all__ = [
    "DEFAULT_ROLE_TIER",
    "GAMIFIED_TO_GENERIC",
    "ModelTier",
    "resolve_model_for_role",
]


class ModelTier(StrEnum):
    """Capability tier for selecting a per-role model.

    Values are the canonical serialized form -- used in TOML, JSONL,
    CLI output, and grep patterns.
    """

    REASONING = "reasoning"
    FAST = "fast"
    BALANCED = "balanced"


# Map of gamified workflow-emitted role strings -> canonical AgentRole.
# Workflows in bonfire.workflows.{standard, research} emit lowercase
# gamified names (scout, knight, ...) into StageSpec.role; the resolver
# normalizes them through this table before tier lookup.
GAMIFIED_TO_GENERIC: Mapping[str, AgentRole] = MappingProxyType({
    "scout":     AgentRole.RESEARCHER,
    "knight":    AgentRole.TESTER,
    "warrior":   AgentRole.IMPLEMENTER,
    "assayer":   AgentRole.VERIFIER,
    "prover":    AgentRole.VERIFIER,   # workflow alias — see §D-CL.1
    "bard":      AgentRole.PUBLISHER,
    "wizard":    AgentRole.REVIEWER,
    "herald":    AgentRole.CLOSER,
    "sage":      AgentRole.SYNTHESIZER,
    "architect": AgentRole.ANALYST,
})


# Default role -> tier mapping. The four roles cited in the ticket
# (researcher=reasoning, tester=fast, implementer=fast, reviewer=reasoning)
# are byte-exact. The other five are inferred from role function:
# - verifier   -> fast      (mechanical pass/fail, no synthesis)
# - publisher  -> fast      (PR scaffolding, structural)
# - closer     -> fast      (merge/announce, mechanical)
# - synthesizer -> reasoning (multi-source synthesis is the reasoning case)
# - analyst    -> reasoning (architectural analysis is reasoning)
DEFAULT_ROLE_TIER: Mapping[AgentRole, ModelTier] = MappingProxyType({
    AgentRole.RESEARCHER:  ModelTier.REASONING,
    AgentRole.TESTER:      ModelTier.FAST,
    AgentRole.IMPLEMENTER: ModelTier.FAST,
    AgentRole.VERIFIER:    ModelTier.FAST,
    AgentRole.PUBLISHER:   ModelTier.FAST,
    AgentRole.REVIEWER:    ModelTier.REASONING,
    AgentRole.CLOSER:      ModelTier.FAST,
    AgentRole.SYNTHESIZER: ModelTier.REASONING,
    AgentRole.ANALYST:     ModelTier.REASONING,
})


def resolve_model_for_role(role: str, settings: BonfireSettings) -> str:
    """Return the user-configured model string for the given agent role.

    Resolution order:
        1. Normalize ``role`` (lowercase, strip).
        2. If the normalized value is a canonical AgentRole, use it.
        3. Else, look up the gamified alias in GAMIFIED_TO_GENERIC.
        4. If neither matches, fall back to ModelTier.BALANCED (no raise).
        5. Look up the tier in DEFAULT_ROLE_TIER for the resolved AgentRole.
        6. Return ``getattr(settings.models, tier.value)``.

    Pure synchronous function. No I/O. No cache. No raise on unknown role.
    """
    normalized = role.strip().lower() if isinstance(role, str) else ""

    # Try canonical AgentRole first.
    try:
        canonical = AgentRole(normalized)
    except ValueError:
        # Fall through to gamified alias lookup.
        canonical = GAMIFIED_TO_GENERIC.get(normalized)  # type: ignore[assignment]

    if canonical is None:
        tier = ModelTier.BALANCED  # graceful default — no raise
    else:
        tier = DEFAULT_ROLE_TIER.get(canonical, ModelTier.BALANCED)

    return getattr(settings.models, tier.value)
```

## D3 — Tier vocabulary (LOCKED)

```python
class ModelTier(StrEnum):
    REASONING = "reasoning"
    FAST = "fast"
    BALANCED = "balanced"
```

**Locked invariants:**

- **Type is `StrEnum`**, not `str | Literal[...]`, not a plain `dataclass`. Matches `AgentRole` precedent (`src/bonfire/agent/roles.py`) — one-string-everywhere discipline.
- **Three values exactly, in this order.** Tests assert `len(ModelTier) == 3` and the iteration order via `list(ModelTier)`.
- **Values are lowercase, ASCII, single-word, no separators.** Same grep-friendly invariants `test_roles.py::test_grep_friendly` enforces for AgentRole.
- **NO ordering semantics.** Tiers are categorical, not ordinal. There is no notion of "REASONING > BALANCED > FAST." Tests forbid relying on `ModelTier.REASONING > ModelTier.FAST` (in fact, StrEnum makes that a string comparison, which is a footgun — explicit test bans it).
- **Module path**: `bonfire.agent.tiers.ModelTier`. Re-exported via `bonfire.agent.ModelTier`. Both import paths must resolve to the same class object (same precedent as TierGate's two-path test).

### Why three tiers + naming-collision audit

Three tiers mirror industry SOP (Opus/Sonnet/Haiku at Anthropic; o1/4o/4o-mini at OpenAI). Two tiers lose the middle ground (the current default `claude-sonnet-4-6` IS that middle); four+ creates a combinatorial assignment problem. The ticket authority is "three tiers." The `tier` word collision with `bonfire.dispatch.tier.TierGate` (commercial tier) is disambiguated by file name (`tiers.py` plural agent-layer vs `tier.py` singular dispatch-layer) and by class name (`ModelTier` prefixed vs `TierGate` suffixed). See §D-CL.3 for the full audit.

## D4 — Default role-tier mapping (LOCKED)

The ticket cites four bindings explicitly:
- `researcher = reasoning`
- `tester = fast`
- `implementer = fast`
- `reviewer = reasoning`

The remaining five canonical roles need a tier. Here is the locked extension with rationale:

| AgentRole | ModelTier | Rationale |
|---|---|---|
| `RESEARCHER` | `REASONING` | **Ticket-cited.** Investigation/synthesis is a reasoning task. |
| `TESTER` | `FAST` | **Ticket-cited.** Writing structural failing tests is mechanical. |
| `IMPLEMENTER` | `FAST` | **Ticket-cited.** Closing tests is mostly mechanical structural code. |
| `VERIFIER` | `FAST` | Mechanical pass/fail check. The "verifier" verb is "look at output, decide." Reasoning value is low. |
| `PUBLISHER` | `FAST` | PR scaffolding (branch, commit, push, write summary). Structural, not reasoning. |
| `REVIEWER` | `REASONING` | **Ticket-cited.** Code review IS the reasoning use case (correctness + design + invariants). |
| `CLOSER` | `FAST` | Merge / announce / close. Pure mechanical. |
| `SYNTHESIZER` | `REASONING` | Multi-source synthesis is the reasoning case (Sage's whole point). |
| `ANALYST` | `REASONING` | Architectural analysis (Architect handler) is the reasoning case. |

**Tier counts:** 5× FAST, 4× REASONING, 0× BALANCED. `BALANCED` is the resolver's FALLBACK for unknown roles (see §D7), not a default-mapped tier — the natural escape hatch when users configure a non-canonical role. The tier mapping is a frozen `MappingProxyType` (stdlib, no third-party dep). Tests assert immutability (`DEFAULT_ROLE_TIER[AgentRole.TESTER] = ...` raises `TypeError`) and exhaustiveness (`set(DEFAULT_ROLE_TIER.keys()) == set(AgentRole)`).

## D5 — `[models]` TOML schema + recommended defaults (LOCKED)

### TOML shape

```toml
[models]
reasoning = "claude-opus-4-7"
fast = "claude-haiku-4-5"
balanced = "claude-sonnet-4-6"
```

### Recommended defaults (per Anthropic catalogue, knowledge cutoff Jan 2026)

- **`reasoning = "claude-opus-4-7"`** — most capable Anthropic model at knowledge cutoff. Opus 4.7 is the latest Opus per project memory (`feedback_wizard_always_opus.md`, `feedback_no_economic_cuts.md`, etc.). Used for review, research synthesis, architectural analysis.
- **`fast = "claude-haiku-4-5"`** — most cost-effective Anthropic model. Used for structural test writing, mechanical code closure, PR scaffolding, merge/announce.
- **`balanced = "claude-sonnet-4-6"`** — current `PipelineConfig.model` default (verified at `src/bonfire/models/config.py:43`). Reusing the existing default for `balanced` means: a `bonfire.toml` with NO `[models]` section continues to dispatch with `claude-sonnet-4-6` as the fallback tier — **byte-identical behaviour to today** for users who never configure `[models]`.

### Pydantic submodel placement + env wiring + BYOK

`ModelsConfig` is a plain `BaseModel` (not `BaseSettings`) per the file's existing convention — only the root `BonfireSettings` is a `BaseSettings`. TOML loading happens for free via `BaseSettings.settings_customise_sources`. `BONFIRE_MODELS__REASONING="..."` works for free via `env_nested_delimiter = "__"` (line 138 of `models/config.py`). All three fields accept ARBITRARY strings — no enum/Literal constraint. The resolver passes whatever the user wrote verbatim to the backend; backend-level validation happens at dispatch time.

## D6 — Per-role tier override (DEFERRED to D-FT B)

v0.1 ships **locked defaults + simple `[models]` user overrides ONLY**. The user-facing override surface in v0.1 is the three string fields under `[models]`. A `[roles.X.tier]` override TOML section is OUT-OF-SCOPE and DEFERRED to D-FT B because (a) the ticket says "default mapping," not "user-config'd"; (b) per-role override opens validation/alias/merge UX issues that are M-sized; (c) users who need per-role override TODAY can use the existing `StageSpec.model_override` escape hatch (`models/plan.py:60`). Rejected TOML syntax `[roles.warrior] tier = "reasoning"` is silently ignored — it is not a parse error, it is just unused (Pydantic's `extra="ignore"` default).

def resolve_model_for_role(role: str, settings: BonfireSettings) -> str:

| `"reviewer"` | REVIEWER | n/a | yes | REASONING | `settings.models.reasoning` |
| `"warrior"` | ValueError | IMPLEMENTER | yes | FAST | `settings.models.fast` |
| `"sage"` | ValueError | SYNTHESIZER | yes | REASONING | `settings.models.reasoning` |
| `"prover"` | ValueError | VERIFIER (alias) | yes | FAST | `settings.models.fast` |
| `"  WARRIOR  "` | (strip+lower) IMPLEMENTER | n/a | yes | FAST | `settings.models.fast` |
| `""` | ValueError | not in map | no | BALANCED (fallback) | `settings.models.balanced` |
| `"alien"` | ValueError | not in map | no | BALANCED (fallback) | `settings.models.balanced` |
| `None` | (str-guard returns "") | not in map | no | BALANCED (fallback) | `settings.models.balanced` |

**All paths return a string. None raise.** This makes the resolver safe to call from any dispatch site without try/except. **A safe-fallback design is mandatory** — the tier resolver is on the hot path of every agent dispatch; an exception here would crash the whole pipeline.

### Why a function, not a class/method

Pure stateless function with two explicit args. No DI need. BON-351's call site in `executor.py` becomes a single line: `model = stage.model_override or resolve_model_for_role(stage.role, self._settings)`.

## D-CL — Constraint Lemmas (sweep-test guards + naming guards)

### D-CL.1 — `prover` is a workflow alias for `verifier`

`src/bonfire/workflows/standard.py:62-67` emits a stage named `prover` with `role="prover"`. There is NO `AgentRole.PROVER` — the canonical generic for "verifier" is `AgentRole.VERIFIER`, and `naming.py:44` maps `verifier → Assayer`. So workflow uses `"prover"` for the same role ADR-001 names `verifier` / `Assayer`. **Memo locks `prover` as a SECOND gamified alias** — both `prover` AND `assayer` resolve to `AgentRole.VERIFIER` in `GAMIFIED_TO_GENERIC`. Workflow rename to `assayer` deferred to D-FT D.

### D-CL.2 — `BonfireSettings` field-set ratchet

`tests/unit/test_config.py::TestBonfireSettingsShape::test_top_level_fields_exactly_these` enforces the exact set `{config_version, bonfire, memory, git, agents}`. Adding `models` REQUIRES updating this test in the Knight RED assertion. NOT fragile — explicit by design (BON-348 §D-CL.2 precedent). The field set is a public contract.

### D-CL.3 — `tier` word collision audit

Three uses of "tier" in v0.1: (1) `bonfire.dispatch.tier.TierGate` — commercial tier free/pro/enterprise (UNRELATED); (2) `bonfire.models.config.PipelineConfig.tier: str = "free"` — same commercial concept (UNRELATED); (3) NEW `bonfire.agent.tiers.ModelTier` — model-capability tier reasoning/fast/balanced. Warrior MUST preserve: file path `tiers.py` plural agent-layer vs `tier.py` singular dispatch-layer; class name `ModelTier` (prefixed) vs `TierGate` (suffixed); test file `test_model_tiers.py` vs `test_dispatch_tier.py`. A purity test in `test_model_tiers.py` asserts `ModelTier is not TierGate`.

### D-CL.4 — `DispatchOptions.role` is BON-351's entry point

`DispatchOptions.role: str` (BON-337) flows from `StageExecutor._dispatch_backend` (`engine/executor.py:271`). BON-351 will swap line 266's `model=envelope.model or self._config.model` for `model = stage.model_override or resolve_model_for_role(stage.role, self._settings)`. **BON-351 passes `stage.role` (gamified workflow value) into the resolver, NOT `options.role` (which doesn't exist yet at DispatchOptions-construction time).** The resolver MUST accept gamified strings — that's why the GAMIFIED_TO_GENERIC alias exists.

### D-CL.5 — Purity test implementation note

The "no-dispatch-import" purity test in `test_resolver.py` MUST use AST inspection, not `sys.modules` (the latter is order-dependent across the test run). Parse `src/bonfire/agent/tiers.py` with `ast.parse`, walk all `ast.ImportFrom` and `ast.Import` nodes, and assert no module path starts with `bonfire.dispatch` or `bonfire.engine`. Pattern matches the existing `test_no_bon_ref_in_src_sweep.py` shape.

### D-CL.6 — TYPE_CHECKING import for `BonfireSettings`

The resolver type-hints `settings: BonfireSettings`. To avoid a circular dependency (`bonfire.agent.tiers` ↔ `bonfire.models.config`), import `BonfireSettings` inside `if TYPE_CHECKING:` (per the `bonfire.protocols` precedent at line 29). The runtime resolver duck-types via `getattr(settings.models, tier.value)`. Tests can pass either a real `BonfireSettings` or a mock with `.models`. NO `# noqa: TC003` — `tiers.py` is not a Pydantic model file (per `feedback_tc003_breaks_pydantic_pytest.md`).

### D-CL.7 — File ordering for `_DEFAULTS`

`_DEFAULTS` (`models/config.py:117-121`) is module-level. `ModelsConfig` must be defined ABOVE `_DEFAULTS` — i.e. after `GitConfig` (line 78), before `_LEGACY_MIGRATIONS` (line 84). MappingProxyType (stdlib) is the canonical immutable-mapping shape for `DEFAULT_ROLE_TIER` and `GAMIFIED_TO_GENERIC` — no third-party `frozendict` dep.
