## C — Winner verdict: **Knight B (Innovation)**

Three reasons, in priority order:

1. **Glob-discovery is load-bearing for memo hygiene at v0.1.** The public
   tree is going to keep accumulating Sage and Knight memos through every
   wave of work. A future memo that drifts from the §D8 prose-vs-list parity
   contract is the EXACT failure mode this cluster guards against. Knight A's
   anchor-style guards the template only — a fresh memo introduces the bug
   class precisely because it is NOT the template. Knight B's
   `SAGE_DIR.glob("*.md")` makes every future memo automatically subject to
   the invariant.
2. **Token-list and parseable-decision contracts age better than substring
   tests.** Knight A's `verdict_words = ("allow", "permitted", "forbid",
   "banned", "discouraged")` set will trip on prose mentions of "allow" in
   any unrelated paragraph; Knight B's machine-parseable inline form
   (`allow: # ---`) is unambiguous and grep-able from CI later. Same for the
   playbook: a token list reads like a contract, a substring test reads like
   a coincidence.
3. **Meta-self-test catches the silent-parametrize-drop failure mode.** The
   single most embarrassing way for a doc-invariant cluster to fail is for a
   parametrize list to be empty, the test to vacuously pass, and nobody to
   notice for six months. B's introspection-based meta-test makes that
   impossible — a future refactor that drops a `test_dft<letter>_*` function
   fails LOUDLY in the next pytest run. This is the right primitive for any
   future cluster-style D-FT bundle.

Soft signal AGAINST B: the test count balloons under parametrize (50
collected vs A's 5). This is mitigated by §E — the test surface for the
bundled PR is locked at 8 distinct functions, not 50 collected items, by
narrowing the parametrize over §D8-bearing memos rather than every memo.

## F — Warrior contract — three doc files to create

### 1. `docs/audit/sage-decisions/_template-sage-memo.md`

**Purpose:** The canonical Sage-memo skeleton. Future Sage memos copy this
file as a starting point. The leading underscore sorts it isolated from
real `bon-NNN-*` decisions and signals "skeleton, not history".

**Required structure:** D1–D10 sections mirroring the shipped Sage-memo
artifacts in this directory. The §D8 section MUST demonstrate prose-vs-list
parity: a prose total of the form `A + B + ... = T tests` AND fenced blocks
listing exactly T distinct `test_*` identifiers. Use `12 + 14 + 21 + 4 =
**51 tests**` (the BON-350 example) verbatim as the demonstration; the
fenced blocks under §D8 list exactly 51 distinct identifiers.

**Required scrubs:** Sample-code blocks (any fenced block with language tag
`python`, `py`, `python3`, or unspecified) MUST NOT contain naked tracker
IDs in the `BON-\d+` shape. Use `ticket-NNN` or `<ticket>` as the
placeholder. Prose outside fenced blocks may cite shipped tickets as
examples (e.g. "see the BON-350 memo for a worked example") — the scrub is
fence-scoped only.

**Forbidden idioms in fenced blocks:** No `# type: ignore[assignment]`. The
canonical pattern is explicit `Optional[T]` annotation on a line above the
`.get()` reassignment. The template's fenced sample MUST demonstrate the
explicit-`Optional` pattern; that demonstration anchors D-FT-B.

### 2. `docs/style.md`

**Purpose:** The single home for v0.1 project-level style decisions. Two
decisions ship in this PR; future decisions append to this file.

**Required structure:**

- A normative section header for section dividers — exactly one of:
  `## Section dividers`, `### Section dividers`, `## Section divider`,
  `### Section divider`, `## Dividers`, or `### Dividers` (case-insensitive
  match).
- The literal `# ---` mentioned at least once in that section.
- A machine-parseable decision line in the inline form
  `allow: # ---` or `forbid: # ---` (lowercased verdict, colon, space, then
  the literal). The verdict for v0.1 is **`allow`** — both `# ---` and
  `# ───` already appear in `src/`; forbidding would force a sweep without
  appetite at v0.1.
- A short rationale paragraph: existing source uses both forms, the
  primary signal is internal consistency within a file, and a future tighter
  ratchet may revisit the decision.

**Required addendum (D-FT-B):** A second normative section
`## Optional annotations` with the rule: prefer explicit `Optional[T]`
annotation on the line above a `.get()` reassignment, over
`# type: ignore[assignment]` suppression. One short python fenced block
demonstrates the preferred idiom.

### 3. `docs/contributor-guide/wizard-pre-stage.md`

**Purpose:** Codify the Wizard pre-merge gate playbook step that the
editable-install lesson memo (`feedback_editable_install_metadata.md`)
encodes. A reviewer running the gate AFTER PR creation, BEFORE merge, must
re-install the Warrior's worktree so `importlib.metadata` returns the
worktree's pyproject version.

**Required tokens (literal substrings):**

- `pip install -e` — the editable-install command.
- `<warrior-worktree>` — the worktree-path placeholder, in angle-bracket
  form (matches markdown placeholder convention).
- `pre-merge gate` — the temporal anchor (BEFORE the gate runs, not
  during, not after).

**Required structure:** A short doc (≤ 200 lines) with sections "When to
run", "Command", "Why this matters", and "What you'll see". The "Command"
section contains a fenced bash block with the literal
`pip install -e <warrior-worktree>` line.

**Cross-reference:** The doc cites this Sage decision memo (relative path)
and links to ADR-001 for module naming context.
