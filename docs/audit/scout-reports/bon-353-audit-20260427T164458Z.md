## Category C — Architecture overview doc for newcomers

The repo has README (consumer-facing), per-module `__init__.py` docstrings, and `docs/audit/sage-decisions/` (decision provenance). It has NO single document that orients a contributor: "what packages exist, what flows through them, where to plug in new behavior."

### **C1 — Recommended path:** `docs/architecture.md`

Sits next to `docs/release-policy.md`, `docs/release-gates.md`, `docs/release-gate-tickets.md`. Newcomers follow the same `docs/` directory; ADRs already live at `docs/adr/`.

### Recommended sections (Warrior writes — Scout sketches)

1. **What Bonfire is** (≈100 words) — pipeline of role-bound agents over a typed event bus. Pull the project tagline from `src/bonfire/__init__.py`.
2. **Module map** — annotated tree of `src/bonfire/`, one sentence per package, mirroring this audit's per-file table. Group as: *core* (engine, dispatch, models, events, protocols), *agents* (agent, handlers, persona, prompt), *integrations* (git, github, knowledge, scan, cost, session, xp), *workflows* (workflows).
3. **Pipeline flow** — narrate one run end-to-end: `bonfire run` → workflow plan → PipelineEngine → per-stage StageExecutor → handler → dispatch backend → event bus → consumers. Cite ADR-001 for naming vocabulary.
4. **Event bus + consumers** — short section: events flow one-way, consumers are decoupled observers, list the four shipped consumers (cost, display, knowledge_ingest, logger) and where to register a new one.
5. **Extension points** — what's pluggable: backends (`AgentBackend` protocol), vault backends (`VaultBackend`), personas (TOML in `persona/builtins/`), workflows (registry).
6. **Gates and quality** — list the six gates in `bonfire.engine.gates`; one sentence each.
7. **Security model** — point at `dispatch/security_hooks.py` + `security_patterns.py`; note that `SecurityHooksConfig` is part of `DispatchOptions`.
8. **Where to read next** — links section.

### Should link

- `docs/release-policy.md`, `docs/release-gates.md`
- `docs/adr/ADR-001-naming-vocabulary.md`
- `docs/audit/sage-decisions/bon-337-unified-sage-2026-04-18.md` (tool policy)
- `docs/audit/sage-decisions/bon-338-unified-sage-2026-04-18.md` (security hooks)
- `docs/audit/sage-decisions/bon-341-sage-*.md` (knowledge layer)
- The README `## What Bonfire Does` section (avoid duplication — link, don't restate)

Length budget for the doc: 250–400 lines. Long enough to orient, short enough to maintain.
